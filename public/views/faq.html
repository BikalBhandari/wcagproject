General Questions

What does this tool actually do?

This tool runs accessibility scans against selected scopes and records findings for alt text, contrast, keyboard access, ARIA, focus handling, target size, link integrity, and outline.

How do I start a new scan?

Go to the Dashboard and click "Start Scan", or use the Scopes tab to run a scan for one scope. You can also create scopes by importing sitemaps.

How does the Coda integration work?

Once you provide a Coda API Token in the Settings, completed results are sent to Coda as a new page with an audit results table. If you do not set a Doc ID, the system uses its default Coda doc.

Why do I only see one issue per element?

The system deduplicates repeated findings and sorts them by severity. It does not merge every overlapping issue into a single element-level finding.

What do I need to configure before running scans?

The main setup items are a Coda API Token, the Vision / Gemini API Key for image analysis, and scan settings like concurrency and timeout. The Coda Doc ID is optional because the app has a default fallback, but you can set your own document in Settings if you want reports written elsewhere.

Reports and Export

How are scan results delivered?

Each completed run generates CSV and PDF reports, and the dashboard and reports views link to the latest exports. If Coda is configured, the same results are also sent there.
