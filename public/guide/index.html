Coda MCP Ingestion Guide

This guide explains how to set up and use the Coda Model Context Protocol (MCP) server to ingest project data (audit reports, task lists, logs) directly into Coda documents using AI-driven prompts.

1 Prerequisites

Coda API Token:

  • Go to your Coda Account Settings.
  • Scroll to API Settings and generate a new API token.
  • Important: Keep this token secure. Paste it into the app’s Settings page for this deployment.
  • The token is stored only on the server instance you configure, in that deployment's local settings file.

2 MCP Configuration

To enable an AI assistant to talk to Coda, add the following to your MCP configuration:

{
  "mcpServers": {
    "Coda": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://coda.io/apis/mcp",
        "--header",
        "Authorization: Bearer YOUR_CODA_API_TOKEN"
      ]
    }
  }
}

3 Strategy: Two-Step Ingestion

For reliable data ingestion, use the Two-Step Atomic Creation pattern. This prevents issues with tables appearing empty while Coda indexes them.

Step A: Create the Container (Page)

"Create a new sub-page in the Coda doc `G__Q5UKx_l` titled 'Report: [Scan Name]'. Return the page URI."

Step B: Create Table with Data (Atomic)

"In the new page `[Page URI]`, create a table named 'Audit Results' with columns [Page URL, Issue, Severity]. Populate it immediately with the data rows provided."

4 Implementation Details

The Coda MCP server uses positional value arrays rather than object mapping for row insertion.

// Example mapping logic
const rows = issues.map(issue => [
    issue.url,
    issue.type,
    issue.severity
]);

5 Troubleshooting

If tables appear but rows are missing:

  • Check Schema: Ensure you are sending rows as an array of arrays.
  • Absolute URIs: The Coda MCP server requires absolute URIs: coda://docs/DocID/pages/PageID.
